# k-029 — Board enhancements

**Done.** All three requests are in.

## 1. Hide/show every column

Every column header (To Do, Doing, Needs OK, Done) is now clickable to collapse or expand it — the same behaviour the Done column had, extended to all of them. A small ▾ / ▸ caret shows the state, and the card count stays visible on the header when collapsed so you still see how many are in there. Each column's collapsed/expanded state is remembered between visits (your previous Done setting carries over automatically).

## 2. Dropping onto a collapsed column

Dragging onto a collapsed column used to be fiddly because there was nothing to drop on. Now, the moment you start dragging a card, any collapsed column reveals a dashed **"Drop here"** strip, so there's always a clear target. Drop onto it and the card moves into that column; the strip disappears again when you finish. You no longer need to expand a column first to move a card into it.

## 3. "Done" button on each card, with a bell

Each card (except those already done) now has a small **✓** button in its top-right corner. Click it and the card moves straight to Done and a bell chimes — like ticking something off in Microsoft To Do. The same bell also plays when you drag a card into the Done column, so completing work always gives the same satisfying ping.

The check button is deliberately separate from the card body, so clicking it completes the card without opening the editor, and it won't interfere with dragging.

## Notes

- The bell uses the browser's built-in audio (no sound files added). Browsers only allow sound after you've interacted with the page, which is always the case here since you're clicking.
- Collapsing a column only hides it on your view — it doesn't change any cards or their status.

## Files changed

- `index.html` — generalised column collapsing to all columns with persisted per-column state (migrating the old Done setting), added a drag-reveal drop zone for collapsed columns, added a per-card Done button, and a bell sound that plays when a card is completed (button or drag-to-Done).

Tested: all scripts pass syntax checks, the stylesheet braces balance, and the page was served locally to confirm the new headers, Done buttons and supporting logic render. Drag/drop and audio are browser-interactive, so a quick click-through on your machine is the final confirmation.
