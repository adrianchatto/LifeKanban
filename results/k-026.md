# k-026 — Backup and restore the board

**Done.** Two new header buttons let you save the board to a file and load it back — including into a Docker instance.

## How to use it

- **⬇ Backup** downloads the whole board as a JSON file named `kanban-backup-YYYY-MM-DD_HHMM.json`. Keep it somewhere safe.
- **⬆ Restore** opens a file picker; choose a backup file and confirm. It replaces the current board with the contents of the file.

### Moving the board to Docker

This is the workflow the card describes:

1. On the desktop board, click **⬇ Backup** and save the file.
2. Start the Docker board (`docker compose up -d --build`) and open it.
3. Click **⬆ Restore**, pick the backup file, confirm — your cards are now in the container (and saved to its data volume).

Because Restore writes through the running app, it works the same on the desktop and in Docker. The downloaded file is also a valid `board.json`, so if you prefer you can drop it straight into the Docker data volume as `board.json` instead of using the Restore button.

## Safety

- Restore always asks for confirmation first, showing how many cards are about to be replaced, and reminds you to take a Backup if unsure. It cannot be undone, so the confirmation is deliberate.
- The file is validated before anything changes: it must be valid JSON containing a `cards` list, otherwise the restore is cancelled with a clear message.
- If the backup is a bare export (missing `next_id` or the project list), those are rebuilt automatically so the board keeps working and new cards get correct IDs.

## A note on result files

The backup contains the board itself (all cards, projects, priorities, due dates, subtasks). The separate result documents in `results/` are not part of the JSON backup — links to them are preserved, but if you move to a fresh Docker instance those files won't be present there unless you also copy the `results/` folder into the volume. For the board data itself, the backup is complete.
