# k-034 — AI API (duplicate of k-033's AI settings)

This card asks for the same thing as the first part of **k-033**: an Admin section to add
an API for the chat assistant, with a provider choice (OpenAI / Anthropic), a pasted API
key, and a model choice. That work is already built and shipped — no separate change was
needed.

## What's in place (delivered under k-033)

- **Admin → AI assistant → Configure** opens a dialog to choose the provider (Anthropic
  or OpenAI), pick a model (presets per provider, e.g. `claude-haiku-4-5`,
  `claude-opus-4-8`, or any custom string), and paste an API key.
- The key is stored only in this browser's `localStorage` — never written to
  `board.json`, so it's never pushed to GitHub.
- When configured, the "Add by chat" assistant uses that model to read your requests and
  falls back to the built-in parser if no key is set or a call fails.

See `results/k-033.md` for the full detail, testing notes, and the decisions made.

No further code change for this card — closing as already done to avoid duplicating the
feature. If you intended k-034 to cover something k-033 didn't, let me know and I'll pick
up the difference.
