# k-043 — Add projects from the chat

**Request:** the ability to add projects from the "Add by chat" assistant, rather than only
through the Projects manager window.

## What I added

The chat assistant now understands **project-creation commands**. Open ✦ Add by chat and
type any of these:

- `add project NEO-015`
- `create projects Home, Work and Admin`
- `new project Marketing`
- `set up project Q3 Planning`
- `please add a new project called Finance`

It adds the project(s) to the board's project list immediately, refreshes the Projects
manager window and the project filter, and confirms what it did (and tells you if a
project already existed). No card is created — it's a pure project-add.

Multiple names work in one go (split on commas, "and", "&", "+", "/"). The help text
inside the chat ("type help") now mentions this too.

## Behaviour notes / guardrails

- Commands are matched **before** the normal card parser, so "add project X" creates a
  project rather than a card.
- A name longer than 40 characters is treated as a task phrase, not a project name, so a
  sentence like "add project plan and send it to Sam" won't accidentally create a giant
  project — it falls through to the card flow.
- The existing behaviour where a `#Tag` on a card auto-creates its project is unchanged.

Tested the command parser against the examples above plus a normal task ("Pay parking
tomorrow", correctly ignored), and the page's inline JavaScript passes a syntax check.

## On "that window where I choose which projects"

That's the **⋮ Projects** manager (top of the board), still there and unchanged — this
change just gives you a second, faster way in via chat. If you actually wanted the chat to
*open* that window on command (e.g. "manage projects"), say so and I'll wire that up too.
