# k-042 — List view, categorised by project

**Request:** a list view that can filter down to projects and see things categorised by
project.

## What I built

Added a third view to the board, alongside **Board** and **Calendar**. The toggle in the
header now reads **Board · List · Calendar**.

The **List** view:

- **Groups every card under its project** — one section per project, project chip and a
  card count in each heading, projects ordered alphabetically.
- **Respects the existing filters.** The "All projects" dropdown and the Me/Claude/Everyone
  filter both apply, so you can either see everything grouped, or narrow to a single
  project and just see that one.
- **Sorts within each project** by status (To Do → Doing → Needs OK → Done), then priority,
  then due date — so the most actionable items sit at the top of each group.
- Each row shows a **status pill**, the title, and the same badges as the board cards
  (priority, Claude assignee, due date, subtask progress, attachments).
- **Rows are interactive:** click a row to open/edit the card, tick ✓ to mark it done, and
  cards waiting on you show a **✓ OK** approve button inline.

Recurring cards stay out of the list (as they do on the board) — they live in the Calendar.
Your chosen view is remembered between visits (it already persisted to `localStorage`).

## How it works under the hood

- New `List` button in `#viewToggle`; new `#listview` container.
- `render()` gained a `view==="list"` branch that calls a new `renderList(cards)` — it
  receives the already-filtered card set, groups by `project`, and reuses the existing
  badge helpers (`projChip`, `dueBadge`, `prioBadge`, etc.) for visual consistency.
- Styling matches the board (light/dark theme variables, project colours).

Validated: the page's inline JavaScript passes a syntax check after the change.

## Files changed

- `index.html` — toggle button, `#listview` container, list-view CSS, `render()` branch,
  and the new `renderList()` function.

Reversible via git. Reload the board (the app auto-refreshes) and the **List** button will
