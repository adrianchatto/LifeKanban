# k-031 — More Kanban enhancements (admin section + dark-mode/colour fixes)

**Done by the Kanban worker, 08/06/2026.** All four asks are in. Refresh the board page to see them (no server restart needed for this card — it's all front-end).

## 1. Admin section

The toolbar was getting crowded, so the utility buttons now live behind a single **⚙ Admin** button in the header. Clicking it opens an Admin panel grouping:

- **Desktop alerts** — enable due/overdue notifications
- **Appearance** — light/dark toggle
- **Back up board** — download the board as JSON
- **Restore board** — replace the board from a backup file
- **Source on GitHub** — the repo link

The header now only carries the things you use constantly: project filter, owner filter, Board/Calendar toggle, Refresh, Projects, Admin and **+ New card**. Each admin item has a one-line description so it's clear what it does. Close with the Close button, by clicking outside the panel, or with Escape.

## 2. Dark-mode buttons that "didn't work very well"

- The modal **Delete** button hardcoded a white background, so it sat as a bright white block in dark mode — now themed to the dark surface.
- Button **hover** was almost invisible in dark mode (a 1% brightness nudge); hover now brightens noticeably so buttons feel responsive.
- "Due soon" and "Due met" date pills had dark text on a dark tint — recoloured to a readable amber/green in dark mode.
- The attachment (📎) and subtask-progress chips now use theme-aware colours instead of a fixed light grey.

## 3. "General is a bit washed out"

The project chip on each card was painting its text colour with a hardcoded dark value via an inline style, which overrode the dark-mode rule — so in dark mode the label was near-unreadable, and "General" (a muted palette colour) looked faint in both themes. Fixed by:

- Using the theme's chip-ink colour for the label, so it's dark text in light mode and light text in dark mode.
- Adding a small **colour dot** to the chip and strengthening the background tint and border, so General (and every project) reads clearly at a glance.

## 4. GitHub link "purple and blue is quite hard to read"

The GitHub link was an `<a>` styled as a button but with no explicit colour, so it fell back to the browser's default link colours — **blue when unvisited, purple once visited** — which clashed badly, especially in dark mode. It now uses the normal text colour (`--ink`) for both visited and unvisited states, with the GitHub mark, sitting cleanly inside the Admin panel.

## Files changed

- `index.html` — Admin button + Admin modal, panel CSS, dark-mode contrast overrides (delete button, due pills, hover, badges), project-chip colour fix with a colour dot, GitHub link colour fix, Escape closes the new panels.

No server change for this card. Verified: JavaScript parses cleanly and every element the script references exists in the markup.

## Note

I kept all the original button IDs and their existing behaviour — only their location moved (into the Admin panel). Nothing about backup/restore/alerts/theme logic changed, so there's no risk to those flows.
