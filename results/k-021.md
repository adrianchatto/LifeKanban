# k-021 — Pomodoro timer on the board

**Done.** A Pomodoro timer now sits in the **bottom-left** of the Kanban board, built into the board's web app (`index.html`). It loads automatically — nothing to install or start.

## What it does

A compact card with three tabs — **Focus / Short / Long** — and a large countdown. Controls underneath:

- **Start / Pause** — runs or holds the current phase.
- **Reset** — sets the current phase back to its full time.
- **Skip** — jumps to the next phase.

It follows the classic Pomodoro cycle: 25-minute focus blocks, a 5-minute short break after each, and a 15-minute long break after every 4th focus block. Completed focus sessions are tallied as 🍅 under the timer. When a phase ends it plays a short chime and (if you've allowed desktop notifications) pops a notification, then automatically rolls into the next phase. While a timer is running, the browser tab title shows the countdown so you can track it from another tab.

The timer is placed bottom-left specifically so it doesn't collide with the "Add by chat" button in the bottom-right.

## Settings

The gear icon (top-right of the widget) opens a small panel to change the focus, short-break, and long-break lengths in minutes. Your settings, the current phase, the time remaining, and your pomodoro count are saved in the browser, so the timer survives a page reload — if it was mid-countdown when you left, it resumes where it should be.

## Notes / decisions

- Standard durations (25 / 5 / 15, long break every 4) are the defaults but are fully adjustable via the gear.
- Breaks auto-start after a focus block, matching how most Pomodoro apps behave. If you'd prefer it to wait for you to press Start each time, that's a one-line change — say the word.
- The chime uses the browser's built-in audio, and notifications reuse the board's existing notification permission (the same 🔔 you can enable in the header). No external services or files were added.

## Files changed

- `index.html` — appended a self-contained Pomodoro module (its own CSS, logic, and widget). The existing board logic was not touched.

Tested: the phase-cycle logic was simulated to confirm the four-focus-then-long-break pattern, the time formatter was checked, the script passed a syntax check, and the page was served locally to confirm the widget renders.
