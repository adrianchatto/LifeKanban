# k-040 — Stop dumping the whole command into the title

**Feedback:** when you give the board a command, the whole sentence lands in the title and
the description stays empty. You want a concise title auto-generated and the full text put
in the description.

## Cause

The board has two quick-add paths:

1. **LLM parse** (`kcLLMParse`) — used when an API key is configured. This already returns
   a proper short title and a separate description.
2. **Regex parse** (`kcParse`) — the no-key fallback. This stripped filler/dates/projects
   and then put *everything left over* into `title`, never touching `description`. That's
   the path producing the long titles.

## What I changed

Patched `kcParse` in `index.html`. After it distils the command, if the result is long
(more than ~10 words or 64 characters) it now:

- generates a **concise title** from the first clause, capped at ~8 words / 56 chars, and
- moves the **full distilled command into the description**.

Short commands (e.g. "Pay for parking") are unchanged — title only, no description.

Tested against several inputs:

| Input | Title | Description |
|---|---|---|
| Pay for parking | Pay for parking | (empty) |
| Chase Acme for the signed SOW because they keep missing the deadline… | Chase Acme for the signed SOW because they | full text |
| Email the team about Friday's release and include the rollback plan… | Email the team about Friday's release and include | full text |

## Honest limitation

The regex fallback can only **truncate** to a sensible short title — it can't truly
summarise ("Chase Acme for signed SOW") the way the LLM path can. If you want clean
distilled titles everywhere, set an API key in the board's AI settings so `kcLLMParse`
handles quick-add; that path already does title + description properly. The regex fix
above is the best behaviour available without a key, and it stops the empty-description
problem regardless.

Separately, the `kanban` SKILL.md already instructs chat sessions to distil a title and
put detail in the description when adding cards from chat — this card fixes the *in-app*
quick-add box, which wasn't following that rule.
