# k-024 — Dark mode

**Done.** The board now has a light/dark toggle. There's a **🌙 Dark / ☀ Light** button in the header — click it to switch. Your choice is remembered between visits.

## What was added

- A theme toggle button in the header. It flips the whole board between light and dark and updates its own label to show what you'll switch to next.
- **Remembered preference.** Your choice is saved in the browser. On first visit (before you've chosen), the board follows your operating system's light/dark setting automatically.
- **No flash on load.** The saved theme is applied before the page paints, so you don't get a flash of the wrong colour scheme when opening or refreshing.
- The browser/PWA theme colour updates to match, so the title bar tints correctly in dark mode too.

## How it works

The board's colours are driven by CSS variables, so dark mode is a clean set of overrides rather than a parallel stylesheet. Switching sets a `data-theme="dark"` attribute on the page, which swaps the palette (backgrounds, text, borders, accents, shadows) and adjusts the surfaces that were previously hard-coded to light colours — cards, columns, the card editor and its inputs, the calendar, the chat assistant, and the Pomodoro timer all follow the theme. Date pickers switch to their dark variant as well.

Everything that was already there — priorities, the chat assistant, the Pomodoro timer, the calendar — works identically in both themes.

## Files changed

- `index.html` — added a dark colour palette and surface overrides via `data-theme="dark"`, a small pre-paint script in the head to apply the saved/OS theme, the header toggle button, and the toggle logic with persistence. A couple of base styles were made theme-aware (input backgrounds) so they read correctly in both modes.

Tested: the page was served locally and confirmed to load with the toggle present; the stylesheet's braces balance and all four script blocks pass syntax checks; and the dark overrides were cross-checked against the hard-coded light surfaces in the CSS so nothing is left glaringly white in dark mode.
