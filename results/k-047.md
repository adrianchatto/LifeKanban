# k-047 — Filter to-do / doing / done on the List page

**Request:** Be able to filter by status (To Do, Doing, Done) on the List view.

**Done.** Added a status filter dropdown to the board toolbar that appears only when the List view is active, and applies live to the list.

## What changed

All changes are in `index.html` (the board UI):

1. **New control** — a `#statusFilter` `<select>` sits next to the existing owner filter, with options: All statuses, To Do, Doing, Needs OK, Done. (I included **Needs OK** as well, since it's a real column on your board and you'll likely want to isolate cards waiting on approval.)

2. **Visibility** — the dropdown is hidden on the Board and Calendar views and only shows on the List view, where status grouping is meaningful. It's shown/hidden automatically inside `render()`.

3. **Filtering logic** — when a status is selected, the List view is filtered to cards whose `status` matches before rendering. "All statuses" shows everything, as before. The filter stacks with the existing project and owner filters.

4. **Wiring** — the dropdown calls `render()` on change, matching how the project and owner filters already behave.

If a filter combination yields nothing, the existing "Nothing to show with the current filters." message appears.

## Verification

Extracted and syntax-checked the main script block with `node --check` — passes clean. The new control reuses the existing filter/render plumbing, so behaviour is consistent with the project and owner filters already in place.

## Notes

- No change to `board.json`, the CLI, or the server — this is purely a front-end view feature.
- The selected status persists while you stay on the List view and is simply hidden (not reset) if you switch to Board/Calendar and back.
