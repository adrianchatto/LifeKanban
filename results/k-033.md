# k-033 — API key for AI chatbot (+ priority and challenge changes)

**Status:** Done. All three requests implemented in the board app itself (`index.html`
and `kanban.py`) and the existing cards migrated. Changes are committed and pushed to
GitHub as part of the worker's board sync.

This card asked for three things. Here's what each one became.

## 1. AI provider settings in Admin

There's a new **AI assistant** row in the Admin panel (⚙ Admin → "AI assistant" →
**Configure**). It opens a small dialog where you can:

- **Choose the provider** — Anthropic or OpenAI (a segmented toggle).
- **Choose the model** — a text box with a dropdown of sensible presets that swap with
  the provider. Anthropic shows `claude-haiku-4-5`, `claude-sonnet-4-6`,
  `claude-opus-4-6`, `claude-opus-4-8`; OpenAI shows `gpt-4o-mini`, `gpt-4o`, `gpt-4.1`,
  `gpt-4.1-mini`, `o4-mini`. You can also type any model string you like, so you're not
  locked to the presets if the names change.
- **Paste an API key** — a password field. **Save**, **Clear**, **Cancel**.

When a key is saved the Admin row shows the live status, e.g. "On — Anthropic ·
claude-haiku-4-5". The **Clear** button removes it again.

**Where the key is stored — and why.** The key is kept only in your browser's
`localStorage` on this device. It is deliberately **not** written to `board.json`, so it
is never committed or pushed to the public GitHub repo. That's the safe place for a
secret; if I'd put it in `board.json` it would have leaked to GitHub on the next sync.
The trade-off is that the key lives per-browser — if you open the board on another
machine you'd paste it again. That's the right call for a secret.

**It actually does something.** When a provider and key are configured, the "Add by
chat" assistant sends your sentence to that model to extract the card fields (title,
project, who, priority, due date, recurrence). If no key is set, or the call fails for
any reason, it silently falls back to the built-in pattern parser — so the chat box
keeps working either way. Calls go direct from the browser to the provider
(Anthropic with the direct-browser-access header; OpenAI with a Bearer token).

One thing to know: browser-to-provider calls can be subject to the provider's CORS and
rate rules, and the key is visible to anything running in that page. For a personal
local tool on your own machine that's fine; just don't host this board on a shared/public
URL with a key saved.

## 2. "None" priority removed; default is now Medium

- The **None** button is gone from the card editor — priority is now High / Medium / Low.
- New cards default to **Medium** everywhere: the editor, the chat assistant, and the
  `kanban.py` CLI (`norm_priority` now maps blank/`none`/`normal` → `medium`, and
  `none` is no longer a valid value).
- **Existing cards were migrated.** A new `python3 kanban.py normalise-priority` command
  set every card that had no priority (or "none") to Medium — 21 of your 27 cards were
  updated; the 3 already marked Low were left alone. (Run via the CLI so the change went
  through the locked read-modify-write path, not a hand-edit of `board.json`.)
- **Board stays readable.** Because everything is Medium now, I made Medium the *silent
  baseline* — the board only shows a priority flag (and coloured edge) for **High** and
  **Low**, the ones that actually deviate. Otherwise every card would carry an identical
  "⚑ Medium" badge, which is just noise. Medium is still selected and visible inside the
  card editor and in the chat draft. If you'd rather see a Medium badge on every card,
  that's a one-line change — say the word.

## 3. The chat assistant now challenges you on project and assignee

When you add a task by chat and you *haven't* said which project it's for or who it's
for, the assistant now stops and asks before adding it:

- **"Which project?"** — a row of buttons for each of your projects (General, Datasite,
  Abenson, MOJ, plus any others).
- **"Who's it for?"** — Me / Claude.

The **Add card** button stays disabled until you've answered the open questions, then it
commits. If you *did* specify them in your sentence (e.g. "…for MOJ, have Claude…" or a
`#tag`), it skips straight to the normal confirm step as before — it only nags when
something's genuinely missing. Priority isn't part of the challenge; it just defaults to
Medium, which matches change #2.
